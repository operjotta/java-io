A linguagem de programação Java, hoje uma das mais influentes e omnipresentes do mundo, tem uma história rica que remonta ao início da década de 1990. Nascida nos laboratórios da Sun Microsystems, a sua criação foi motivada por um desejo de superar os desafios da programação em ambientes heterogéneos, especialmente no crescente mundo dos dispositivos eletrónicos de consumo doméstico.

Tudo começou em 1991, quando um pequeno grupo de engenheiros da Sun, liderado por James Gosling, Michael Sheridan e Patrick Naughton, formou um projeto secreto conhecido como "The Green Project". O objetivo inicial não era criar uma linguagem para computadores pessoais, mas sim uma tecnologia para programar eletrodomésticos inteligentes e dispositivos portáteis. A ideia era que estes dispositivos precisariam de um software que pudesse ser executado em diferentes arquiteturas de hardware, algo que as linguagens C e C++ da época tornavam notoriamente difícil.

A primeira linguagem desenvolvida no âmbito deste projeto foi inicialmente chamada "Oak" por James Gosling, em homenagem a uma árvore de carvalho que ele via da janela do seu escritório (Oak é carvalho em inglês). A linguagem Oak foi concebida para ser simples, orientada a objetos, robusta, segura, independente de arquitetura, portátil, de alto desempenho, multithreaded (ou seja, executa várias instruções simultaneamente) e dinâmica. Muitos destes princípios fundadores continuam a ser pilares do Java até hoje. A independência de arquitetura era crucial; a equipa queria uma forma de o código compilado poder ser executado em qualquer processador, independentemente do sistema operativo subjacente.

O foco do "The Green Project" acabou por mudar com o advento da "World Wide Web". Em 1994, a equipa percebeu que a internet era um ambiente ainda mais heterogéneo do que o dos eletrodomésticos. A capacidade do Oak de criar código portátil ("bytecode") que pudesse ser executado numa máquina virtual (a Java Virtual Machine - JVM) tornava-o ideal para aplicações web interativas, que eram na altura maioritariamente estáticas.

Em 1995, a linguagem Oak foi renomeada para "Java". O nome, segundo a lenda, foi escolhido durante uma sessão de brainstorming numa cafetaria, inspirada no café que os engenheiros consumiam. O lançamento oficial do Java e do seu navegador compatível, o HotJava, ocorreu em maio de 1995. O slogan "Write Once, Run Anywhere" (WORA) tornou-se o mantra do Java, com uma promessa e portabilidade sem precedentes que o mercado ansiava.

A adoção inicial do Java foi impulsionada pela sua capacidade de criar applets – pequenas aplicações que podiam ser incorporadas em páginas web e executadas nos navegadores dos utilizadores. Isto trouxe dinamismo e interatividade a uma web que era, na altura, predominantemente estática. Netscape Navigator, o navegador dominante da época, rapidamente integrou suporte para applets Java, solidificando a sua posição inicial.

O lançamento do Java Development Kit (JDK) 1.0 em janeiro de 1996 marcou um passo crucial. Incluía a Java Virtual Machine (JVM), o compilador Java e as bibliotecas padrão, fornecendo aos desenvolvedores as ferramentas necessárias para construir aplicações Java. Seguiram-se rapidamente versões melhoradas, com o JDK 1.1 em 1997, que introduziu funcionalidades importantes como JavaBeans, JDBC (Java Database Connectivity) e RMI (Remote Method Invocation).

O final da década de 1990 viu o Java expandir-se para além das applets. O lançamento do J2EE (Java 2 Platform, Enterprise Edition) – mais tarde Java EE e agora Jakarta EE – estabeleceu o Java como uma força dominante no desenvolvimento de aplicações empresariais do lado do servidor. J2SE (Java 2 Platform, Standard Edition) tornou-se a base para aplicações desktop e de propósito geral, enquanto J2ME (Java 2 Platform, Micro Edition) visava dispositivos móveis e embarcados. Esta estratégia de "plataformas" segmentadas ajudou o Java a crescer em diversos mercados.

O início dos anos 2000 trouxe mais inovações e desafios. O Java atravessou várias batalhas legais, notavelmente com a Microsoft sobre a implementação da JVM. No entanto, continuou a evoluir com lançamentos regulares do J2SE (renomeado para Java SE com a versão 5 em 2004). Java SE 5 (codinome "Tiger") foi um lançamento particularmente significativo, introduzindo genéricos, anotações, autoboxing/unboxing e um loop for melhorado, funcionalidades que aumentaram significativamente a produtividade do desenvolvedor.

Outras versões notáveis do Java SE incluíram a versão 6 (2006), que melhorou o desempenho e o suporte a scripts, e a versão 7 (2011), que adicionou funcionalidades como o operador diamond e try-with-resources.

Um dos desenvolvimentos mais impactantes na história recente do Java foi a sua aquisição pela Oracle Corporation em 2010, como parte da compra da Sun Microsystems. Esta aquisição gerou alguma incerteza na comunidade Java, particularmente em relação ao futuro do código aberto e à gestão do ecossistema. A Oracle continuou a desenvolver o Java, mas também esteve envolvida em litígios de alto perfil, como o processo contra a Google sobre o uso de APIs Java no sistema operativo Android.

Falando em Android, este é outro pilar fundamental do ecossistema Java moderno. Lançado em 2007, o sistema operativo móvel da Google utiliza a linguagem Java (e a sua própria máquina virtual, a Dalvik, posteriormente substituída pela ART) como a linguagem primária para o desenvolvimento de aplicações. Isto abriu um vasto novo mercado para desenvolvedores Java, solidificando a sua relevância na era mobile.

Nos últimos anos, a Oracle mudou o modelo de lançamento do Java para um ciclo mais rápido, com novas versões a cada seis meses (a partir do Java 9 em 2017). Lançamentos de Suporte de Longo Prazo (LTS) ocorrem a cada poucos anos (como Java 11, 17 e 21), fornecendo versões estáveis para implantações empresariais. Esta cadência mais rápida permite que novas funcionalidades cheguem aos desenvolvedores mais cedo.

A comunidade Java continua vibrante e ativa, impulsionando a inovação através do Java Community Process (JCP), que governa o desenvolvimento da linguagem e das plataformas. Projetos de código aberto como o OpenJDK (a implementação de referência do Java SE) desempenham um papel crucial.

Hoje, o Java é mais do que apenas uma linguagem; é uma plataforma vasta e um ecossistema robusto. É a espinha dorsal de inúmeras aplicações empresariais, sistemas back-end em larga escala, aplicações Android, sistemas embarcados e muito mais. Frameworks populares como Spring e Jakarta EE continuam a evoluir, facilitando o desenvolvimento de aplicações complexas. Ferramentas como Maven e Gradle simplificam a gestão de projetos.

Apesar do surgimento de novas linguagens e tecnologias, o Java mantém a sua relevância devido à sua maturidade, desempenho, segurança, vastas bibliotecas, forte suporte da comunidade e ao princípio subjacente do WORA que continua a ser incrivelmente valioso num mundo cada vez mais distribuído e heterogéneo. A história do Java é uma jornada de adaptação e evolução, desde o seu início humilde como uma linguagem para eletrodomésticos até se tornar uma das forças mais poderosas e duradouras no mundo da tecnologia da informação. A sua longevidade é um testemunho do design robusto e da capacidade de se adaptar a novas tendências e necessidades do mercado.
